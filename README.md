# Tic-Tac-Toe Multiplayer

\# Tic-Tac-Toe Multiplayer



A full-stack real-time multiplayer Tic-Tac-Toe game built with HTML, CSS, JavaScript, FastAPI, and WebSockets.



\## Features



\### Local Multiplayer



\* Two-player local gameplay

\* Turn-based game logic

\* Winner and draw detection

\* Score tracking (X Wins, O Wins, Draws)

\* Rematch functionality

\* Alternating starting player between rounds



\### Online Multiplayer



\* Create and join private game rooms

\* Real-time lobby synchronization using WebSockets

\* Live game state synchronization

\* Turn validation on the server

\* Real-time move updates

\* Persistent score tracking during a session

\* Rematch voting system

\* Player disconnect handling



\### User Experience



\* Custom username selection

\* Room code sharing

\* Copy room code functionality

\* Responsive dark-themed interface

\* Deployed frontend and backend

\## How It Works



\### Local Multiplayer



The local game uses client-side game state management with JavaScript. Players alternate turns, and the application checks for winning combinations or draw conditions after every move.



\### Online Multiplayer



The online mode uses FastAPI WebSocket endpoints to maintain real-time communication between players.



Workflow:



1\. Player creates a room.

2\. Room code is generated by the backend.

3\. Second player joins using the room code.

4\. Lobby synchronizes player information.

5\. Host starts the game.

6\. Moves are validated by the server.

7\. Board updates are broadcast to both players.

8\. Players can request rematches after a game ends.
